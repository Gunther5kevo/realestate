# NestIQ

A real estate platform for the Kenyan market. Users can browse and book property viewings, agents can list and manage properties, and admins oversee the entire platform through a web dashboard.

---

## Repository Structure

```
realestate/
  nestiq_flutter/      Flutter mobile app (iOS and Android)
  admin_dashboard/     React web app for platform administration
```

---

## Flutter App

The mobile app built with Flutter and Firebase.

**Features**
- Browse and search property listings
- Filter by type, price, bedrooms, location
- Book property viewings with M-Pesa or card payment
- View agent profiles and contact via call or WhatsApp
- Save properties to a personal list
- Interactive map view of listings
- Agent dashboard for managing own listings and bookings

**Tech**
- Flutter with Riverpod for state management
- Firebase Auth, Firestore, Storage, Cloud Functions
- M-Pesa Daraja API via Cloud Functions
- GoRouter for navigation

**Setup**

1. Install Flutter SDK
2. Clone the repo and navigate to `nestiq_flutter/`
3. Run `flutter pub get`
4. Add your `google-services.json` (Android) and `GoogleService-Info.plist` (iOS) from your Firebase project
5. Add your `lib/firebase_options.dart` generated by FlutterFire CLI
6. Run `flutter run`

## Admin Dashboard

A web-based admin panel built with React and Vite.

**Features**
- Login with admin role check
- Dashboard with platform stats and charts
- Listings management — approve, reject, feature, suspend
- User management — search, filter, upgrade to agent, suspend
- Agent management — verify or remove verification
- Bookings overview — confirm, complete, cancel, delete
- Transactions — payment history with M-Pesa and card filters, CSV export
- Push notifications — send to all users, all agents, or a specific user

**Tech**
- React 18, Vite, Tailwind CSS
- Firebase SDK (same project as the mobile app)
- Recharts for analytics charts
- PapaParse for CSV export

**Setup**

1. Navigate to `admin_dashboard/`
2. Run `npm install`
3. Open `src/firebase/config.js` and replace the placeholder values with your Firebase web app credentials
4. Run `npm run dev`
5. Open `http://localhost:3000`

The account you log in with must have `role: "admin"` set on its Firestore user document. Any other role is blocked at login.

## Firebase Setup

Both the Flutter app and the admin dashboard connect to the same Firebase project.

Collections used:

| Collection | Description |
|---|---|
| `users` | All registered accounts with role field |
| `properties` | Property listings |
| `agents` | Agent profiles |
| `bookings` | Property viewing bookings |
| `transactions` | Payment records |
| `notifications` | Sent push notification history |
| `notificationQueue` | Queue processed by Cloud Functions for FCM dispatch |
| `activity` | Platform activity feed shown on the dashboard |

Firestore rules are in `admin_dashboard/firestore.rules`. Deploy them with:

```bash
firebase deploy --only firestore:rules
```
